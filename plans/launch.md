## Launch angle

**Who:** TypeScript/Node developers shipping repos that AI agents (Claude Code, Codex, Cursor)
will read and act on.

**Job:** Instantly know if your repo is ready for an AI agent to onboard and work in it
without hand-holding — before you share it, open-source it, or hand it to a teammate's agent.

**Why now:** Every team is now pairing humans with AI coding agents. The missing primitive
is a fast, CI-friendly score that tells you whether your repo has the basics an agent needs:
README, AGENTS.md, predictable scripts, conventional commits, docs, and a clear entrypoint.

**Differentiator:** One command. No config. No sign-up. Zero dependencies at runtime.
Runs in CI with an exit-code gate (`--min-score 75`). Produces text, markdown, or JSON.

**Tagline options:**

- "Score your repo hygiene in one command."
- "Is your repo agent-ready? Find out in 3 seconds."
- "The `eslint` for agent onboarding hygiene."

### Product Hunt listing

**Tagline:** Score your repo's AI-agent readiness in one command.

**Description:**

```
agent-hygiene-linter is a zero-config CLI that scans any repository and gives
it a hygiene score from 0–100. It checks the seven signals that determine
whether an AI coding agent (Claude Code, Codex, Cursor) can onboard and work
in your repo without hand-holding:

• README at root
• AGENTS.md or CLAUDE.md with agent instructions
• docs/ directory with at least one Markdown file
• CHANGELOG.md or release notes
• Standard npm scripts: build, test, lint, ci
• Conventional Commits ratio (last 25 commits)
• Obvious entrypoint: src/index.ts, index.ts, or main.ts

Run it locally, gate your CI with --min-score 75, or pipe --format json into
a monitoring dashboard. MIT licensed. Zero runtime dependencies.
```

**Topics:** Developer Tools, Open Source, CLI, AI Tools

## Share loops

**The score as a badge:** Add a hygiene score badge to your repo README once
a CI integration is in place. Other devs will click it. Example framing:
"Hygiene: 98/100" as a terminal-output badge via shields.io or a custom endpoint.

**Before/after demo:** Record a terminal session (asciinema or vhs) showing
a low-hygiene repo improving from 46 → 100 after adding README + AGENTS.md +
a changelog. This is the most shareable artifact for X and LinkedIn.

**"What's your score?" reply loop:** Post output of running it on well-known
open-source repos (with permission/attribution). Scores above 90 and below 50
are both interesting for different reasons.

**Blog post format:** "7 things your repo needs before you hand it to an AI
agent" — use the checks as the structure. Each section is one check, with
a before/after code block. This format travels well on dev.to and HN.

**Programmatic API for integrators:** The `scanRepository()` export lets
dashboard tools, GitHub Apps, and CLI wrappers embed the scanner. This
creates a downstream integration surface.

## Risks and assumptions

| Risk                                 | Mitigation                                                     |
| ------------------------------------ | -------------------------------------------------------------- |
| "Why not just check manually?"       | The CI gate + JSON output is the answer; document it upfront   |
| Score feels arbitrary                | `docs/checks.md` explains every deduction; link it from README |
| Node-only limits reach               | Document that it works on any repo, not just Node projects     |
| No GIF/demo makes it harder to share | Record one before posting to HN or Product Hunt                |
| npm publish not yet live             | Release to npm before posting anywhere public                  |

Assumptions:

- "Agent hygiene" as a term will resonate with the audience already using Claude Code/Codex.
  If it doesn't, "repo readiness score" is the fallback framing.
- The --min-score CI gate is the primary adoption vector, not the one-off scan.
